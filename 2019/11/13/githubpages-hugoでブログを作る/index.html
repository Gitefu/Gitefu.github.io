githubPages+hugoでブログを作る

By tanaka / 2019-11-13 / In categories その他

githubPages, hugo, Web関連

はじめに

　私はmacで作りました。
WSLを使えばWindows人もhugoのインストール以外は同じ手順で良いはずです。(他はよく分からぬ)
　あと最近(?)公式がjekyllを推しています。使いやすそうですね。hugoに拘らない人はそっちを調べる方が吉かも？しれません。

このサイトの構築

　このサイトはgithubPagesをhugoと組み合わせて作りました。hugoの使い方に戸惑って2日消費しましたが、合計3日程度で作れました。
　忘れないうちにと思って慌てて書いたので鵜呑みにはしない方が良いでしょう。他の記事も参考にして、参考の1つにぜひ。

導入は簡単

　hugoを使うメリットとしては、
　　・導入が楽
　　・テンプレートが用意されているが、
　　　　cssを使って簡単にカスタマイズできる
　　・マークダウン方式で記事追加が簡単

どういう手順か

　ここでは導入の話をします。テンプレの弄り方とかはまた今度。
　　① ターミナルを知る
　　② hugoを導入する
　　③ hugoのテーマを選ぶ
　　④ gitをちょっと触る
　　⑤ githubPagesを用意する
　　⑥ hugoで作ったサイトをgitを用いて公開する

① ターミナルを知る

　なるべくはじめての人向けに作ろうと思いますが、最低限 「ディレクトリ」「cdコマンド」「lsコマンド」「mkdirコマンド」 等を知っている前提で進めます。分からない人はググってから来てください。
　まずはサイトを構築するためのディレクトリを用意して、そこまでターミナルで移動(cd)しておいて下さい。

② hugoの導入

　hugoがそもそも何かというと静的サイトジェネレーターってやつです。みなさん適当なwebサイトのソースコードって見たことありますか？こんな感じです。 　んー、これは書きにくい、、ミスりそうじゃありません？
　そこで使うのが静的サイトジェネレーターってやつです。下の画像は同じ記事のmd(マークダウン)形式のファイルです。 圧倒的に見やすいので絶対ミスらねえ(フラグ)！
静的サイトジェネレーターは先ほどの画像のようなmd(マークダウン)形式のファイルをうまいことして、複雑なhtml形式に変換してくれます。
　では実際に静的サイトジェネレーターである「hugo」 をインストールしましょう。

1  $brew install hugo

もしエラー起きたら多分Homebrew関連？ですかね。エラーがなければインストールが出来ました。
　次にサイトの準備をします。
hugoはテンプレートや様々なファイルを基にmdファイルをhtml形式に変換してくれます。その「様々なファイル」を次のコマンドで用意しましょう。

1  $hugo new site ファイル名

この「ファイル名」はサイト名とかとは一切関係ありません、自由につけて下さい。このコマンドを実行したらlsコマンドを打って下さい。先ほど入力したファイル名のディレクトリが作成されていたらOKです。
　作成できていたら、そこにcdしましょう。

③ hugoのテーマを選ぶ

　テーマを探します。「hugo theme」とか「hugo テーマ」で調べてみて下さい。色々ありますね。できたら「exampleSite」的な例を含んだファイルを持っている物にして下さい。(以後このファイルをexampleファイルと呼びます) この画像のURLはこちら
　気に入ったテーマのファイルをzipとかでインストールします。そして解凍するとそのテーマの名前のファイルが出てくると思います。そのファイルをコピーして、②で作成したディレクトリのthemesの中にペーストして下さい。
　その後、今ペーストしたファイルの中のexampleファイルの中身をコピーして、②で作成したディレクトリにペースト(上書き) して下さい。 ここまでくれば、②で作成したディレクトリに移動したのちに以下のコマンドを実行してみて下さい。

1  $hugo server -w

を実行して、ウェブブラウザでlocalhost:1313/ を開いてみて下さい。止めるときはctrl+Cでできます。
ここで失敗した人はもう一回全部消してやり直すのが良いかもしれません。(自分の場合はテーマを別のものに変えたらいい感じでいきました)
　よかった、これでサイトのデータを作れるようになりましたね(強制)。
　・・・ガバガバでごめんな、、許して、、。

④ gitの軽い説明

　次は作ったデータを公開するためにgitに慣れましょう。個人のブログを想定して話をします。やることは3つ 「addして、commitして、pushする」 だけ。例えばfirst.md内のurlを変更した場合、こんな感じです。

1  $git add first.md
2  $git commit -m "fixed URL in first.md"
3  $git push

　本来、他にも色々あるのでしょうが、よく分からんので最低限をここに書きます。
　後ろから順に解説します。まず「push」 というのは、 「ローカルで変更した点を基に、インターネット上のデータも同様に変更する」 的な行為と思って下さい(ブランチとか初心者には複雑なので知らんぷりで今はOKでしょう)。
　次にcommitの説明をしましょ。先ほど話した「ローカル(PC)で変更した点」をローカル上で記録する行為をcommitすると言います。この行為はインターネット上にupするわけではありません。あくまで自身のパソコン上でまとめるだけです。”-m”でその変更自体に名前をつけられます。上の例では”first.md内のurlを変更した”という意味で”fixed URL in first.md”としました。
　最後にaddについて話します。addは「変更を追加する行為」 と思って下さい。上の例ではfirst.mdの中身を変更したのでfirst.mdをaddしました。逆に削除する場合は”$git rm first.md”でできるはずです。

⑤ githubPagesを用意する

　まずgithubに登録しましょ。
　　https://github.com/
　次に画像の緑のNewボタンをクリック！
　次にリポジトリ名を登録します。”ユーザー名.github.io”でにしましょう。私のユーザー名は「Gitefu」なので以下の画像のようになります。
そしたらリポジトリ完成です。
　次にローカルに環境を構築します。②で作成したディレクトリをターミナルで開いて下さい。そこでまずgithubの設定をしましょう。

1  $git config --global user.name "[ユーザー名]"
2  $git config --global user.email "[登録したメールアドレス]"

これでユーザーは変更されました。次に公開するために先ほど作成したリポジトリとローカルのディレクトリを紐付けしていきましょう。
　まずはpublicへ移動しましょう。(publicの中身を公開したいので)

1  $cd public

次にここをリポジトリと紐付けしましょう。

1  $git clone [URL] #今回はhttps://github.com/[ユーザー名]/[ユーザー名].github.io.git

これで準備できました。

⑥ hugoで作ったサイトをgitを用いて公開する

　まずは②で作成したディレクトリに移動しましょう。 　次には記事の追加をしてみましょう。

1  $hugo new posts/記事名.md #たまにpost/記事名.mdの場合もあるので要確認

これでmdファイルが作成されました。ちなみに勝手にmdファイルを作ったり移動したりすると、ジェネレーターはうまく変換してくれません！絶対”hugo new”を使いましょう！
　作成したmdファイルを開いてexampleファイルの中の記事を基に好きに変更して保存しましょう。
もし日本語を適応したいのであれば、現在のディレクトリのconfig.tomlに以下を追加すれば良いかと思います。

1  defaultContentLanguage = "jp"

記事を追加したら①のようにして確認してみましょう。

1  $hugo server -w

どうですかね。。？できてます。。？
んでできたらCtrl+Cで止めて下さい。
　止めたら次に改めてジェネレータにhtmlに変換してもらいましょう。

1  $hugo --cleanDestinationDir

これでpublicファイルに変更されたファイルが書き起こされたはずです。
　最後にpublicに移動してaddしてcommitしてpushすればいいはずです。(削除等は今後慣れたらうまくやって下さい。)

1  $cd public
2  $git add *
3  $git commit -m "add new page"
4  $git push

これで多分うまくできました。
私（ユーザー名Gitefu）の場合はhttps://gitefu.github.io/で開きます。皆さんの場合は[https://[ユーザー名].github.io/]かと思います。

・・・できました？
